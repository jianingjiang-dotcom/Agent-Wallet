# PRD: 钱包恢复流程 V2

---

## 1. 背景与目标

### 1.1 业务背景

Cobo Agentic Wallet 采用 MPC 架构，用户设备持有密钥分片。更换设备后需恢复分片才能签名（转账、审批 Pact）。恢复前钱包处于"未激活"状态——可查看资产，不可操作。

### 1.2 目标

1. 合并 iCloud / Google Drive 为统一的"云端恢复"入口
2. 恢复流程中展示可恢复 / 不可恢复的钱包明细
3. 支持通过重新配对（reshare）恢复分片
4. 完善"未激活"状态的全局提示与操作引导
5. 关键操作增加安全确认（生物识别、TSS Node ID 校验）

---

## 2. 术语定义

| 术语 | 定义 |
|------|------|
| **密钥分片** | MPC 架构下用户设备持有的私钥碎片，与服务端分片配合完成签名 |
| **未激活** | 当前设备未持有可用分片，钱包只读 |
| **已激活** | 当前设备持有分片，可正常签名 |
| **Reshare（重新配对）** | 通过 Agent 配对码重新生成分片 |
| **TSS Node ID** | 分片唯一标识，reshare 后会变更 |

---

## 3. 恢复方式选择

用户进入恢复页面后，看到两种恢复方式：

| 方式 | 说明 |
|------|------|
| **云端恢复** | 从 iCloud 或 Google Drive 备份恢复（合并原 2 个入口） |
| **本地文件恢复** | 导入 .backup 备份文件恢复 |

页面底部有"跳过恢复"入口（仅在首页显示，进入具体恢复子流程后隐藏）。

## 6. 重新配对（Reshare）恢复

### 6.1 入口

重新配对不在恢复方式选择页中，入口分布在：

| 入口 | 位置 | 原型 |
|------|------|------|
| 未激活首页 banner | 钱包首页顶部 | 见 §8 原型 |
| 钱包管理"激活"按钮 | 未激活钱包卡片右侧 | 见 §9 原型 |

### 6.2 流程

```
输入配对码 → 确认钱包信息 → 密钥分片生成 → 备份更新
```
